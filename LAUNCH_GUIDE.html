Configure Supabase Auth
Set up email auth, redirect URLs, and the new user trigger.

6a — Enable email auth:

  • Supabase Dashboard → Authentication → Providers
  • Ensure Email provider is enabled
  • Set Confirm email to ON (important for .edu verification flow)

6b — Add redirect URLs (required for magic link emails):

  • Supabase Dashboard → Authentication → URL Configuration
  • Set Site URL to your Vercel URL (you'll get this in Step 11 — come back and add it)
  • Add to Redirect URLs: https://YOUR-DOMAIN.vercel.app/**
  • Also add for local dev: http://localhost:3000/**

6c — Auto-create profile row on signup:

Run this SQL in the Supabase SQL Editor. It creates a profiles row automatically whenever a new user signs up via Supabase Auth.

SQL EDITOR — Paste and run this
-- Function: auto-create profile row on new auth.users insert
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO public.profiles (id, role)
  VALUES (
    NEW.id,
    COALESCE((NEW.raw_user_meta_data->>'role')::public.user_role, 'student')
  );
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE OR REPLACE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();

-- Function: auto-create student_profile row for student signups
CREATE OR REPLACE FUNCTION public.handle_new_student_profile()
RETURNS TRIGGER AS $$
BEGIN
  IF NEW.role = 'student' THEN
    INSERT INTO public.student_profiles (user_id)
    VALUES (NEW.id)
    ON CONFLICT (user_id) DO NOTHING;
  END IF;
  IF NEW.role = 'vendor' THEN
    INSERT INTO public.vendor_profiles (user_id, business_name, city)
    VALUES (NEW.id, 'My Business', 'City')
    ON CONFLICT (user_id) DO NOTHING;
  END IF;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE OR REPLACE TRIGGER on_profile_created
  AFTER INSERT ON public.profiles
  FOR EACH ROW EXECUTE FUNCTION public.handle_new_student_profile();

Create Storage Buckets
For student ID uploads and offer images.
  • Supabase Dashboard → Storage → New bucket
  • Create bucket: verification-docs → set to Private (ID documents must be private)
  • Create bucket: offer-images → set to Public (offer photos are public)
  • Create bucket: vendor-logos → set to Public
SQL EDITOR — Storage RLS policies
-- Students can upload to their own verification-docs folder
CREATE POLICY "Students upload own verification docs"
  ON storage.objects FOR INSERT
  WITH CHECK (
    bucket_id = 'verification-docs'
    AND auth.uid()::text = (storage.foldername(name))[1]
  );

-- Admins can read all verification docs
CREATE POLICY "Admins read verification docs"
  ON storage.objects FOR SELECT
  USING (
    bucket_id = 'verification-docs'
    AND EXISTS (
      SELECT 1 FROM public.profiles
      WHERE id = auth.uid() AND role = 'admin'
    )
  );

-- Vendors can upload offer images
CREATE POLICY "Vendors upload offer images"
  ON storage.objects FOR INSERT
  WITH CHECK (bucket_id = 'offer-images' AND auth.role() = 'authenticated');

-- Offer images are publicly readable
CREATE POLICY "Offer images public read"
  ON storage.objects FOR SELECT
  USING (bucket_id = 'offer-images');

Set Up the Cron Job
Auto-expires stale voucher codes every hour.
  • Supabase Dashboard → Database → Extensions → search for pg_cron → enable it
  • Go to SQL Editor and run:
SQL EDITOR
-- Run code expiry every hour
SELECT cron.schedule(
  'expire-redemption-codes',
  '0 * * * *',
  'SELECT public.expire_stale_redemptions();'
);

-- Verify it was created
SELECT * FROM cron.job;

Post-Deploy Checks
Go through this before sharing the link with anyone.
  • Landing page loads — visit your Vercel URL, see the hero section
  • Student signup works — register with a real .edu email, get a verification email
  • Verification email arrives — check spam. Click the link. Profile updates to verified
  • Student dashboard loads — see the offer grid (empty until you create offers)
  • Vendor signup works — register a vendor account
  • Create a test offer — log in as vendor → Vendor Dashboard → Create Offer
  • Student can see offer — log in as student → browse → offer appears
  • Claim flow works — student clicks "Get Voucher" → QR code appears
  • Vendor confirmation works — vendor clicks "Scan Voucher" → enters code → sees green ✓ screen
  • Redemption count updated — vendor dashboard shows 1 redemption
  • Go back to Step 6 and add your Vercel URL to Supabase Auth redirect URLs
  • Promote yourself to admin — run the SQL below so you can approve student ID uploads
SQL EDITOR — Promote yourself to admin
-- Run this after signing up with your own account
UPDATE public.profiles
SET role = 'admin'
WHERE id = (
  SELECT id FROM auth.users
  WHERE email = 'your@email.com'
  LIMIT 1
);

Connect Looker Studio
Build your vendor ROI dashboards — no ETL needed.
📊
Supabase exposes a direct PostgreSQL connection. Looker Studio has a native PostgreSQL connector. No middleware, no export step — live data.

Get your database credentials:

  • Supabase Dashboard → Project Settings → Database
  • Copy: Host, Database name, Port (5432), User (postgres)
  • Your password is the one you set in Step 4

Create a read-only analytics user (recommended — never give Looker your main password):

SQL EDITOR — Create read-only Looker user
-- Create a read-only role for Looker Studio
CREATE ROLE looker_readonly LOGIN PASSWORD 'PickAStrongPassword123!';

GRANT USAGE ON SCHEMA public TO looker_readonly;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO looker_readonly;
GRANT SELECT ON ALL SEQUENCES IN SCHEMA public TO looker_readonly;

-- Allow future tables to be readable too
ALTER DEFAULT PRIVILEGES IN SCHEMA public
  GRANT SELECT ON TABLES TO looker_readonly;

Connect in Looker Studio:

  • Go to lookerstudio.google.com → Create → Data Source
  • Search for PostgreSQL connector
  • Enter: Host = db.YOUR_PROJECT_REF.supabase.co
  • Port: 5432, Database: postgres, Username: looker_readonly
  • Enable SSL
  • Click Authenticate → you'll see all your tables and views
